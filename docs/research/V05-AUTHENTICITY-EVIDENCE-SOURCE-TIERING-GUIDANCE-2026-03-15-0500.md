# V05 authenticity evidence-source tiering guidance — 2026-03-15 05:00 UTC

## Question
Should evidence-source categories be standardized/tiered so weak sources are not counted as “independent” for authenticity completeness?

## Research signal summary
External search quality was mixed/noisy, but a stable cross-domain pattern emerged from provenance and reproducibility ecosystems:
- provenance systems emphasize attestable step/source identity,
- evidence management emphasizes chain/traceability,
- benchmark reproducibility emphasizes independent artifact components (code/data/env/logs), not one self-reported channel.

## Decision
**Yes — standardize source categories and independence tiers now.**

Current boolean (`non-metadata source exists`) is too weak and can be gamed by low-value pseudo-independent labels.

## Proposed source-tier model (for next build slices)

### Tier 0 — Self-asserted (non-independent)
- Examples: `metadata.sourceOfMove`, free-form operator annotation.
- Rule: contributes to coverage, **never** to independence.

### Tier 1 — Derived local artifacts (conditionally independent)
- Examples: `checkpoint.results`, parsed runner logs, deterministic local trace output.
- Rule: counts toward independence only when generated by a pipeline path separate from move producer and linked to run id.

### Tier 2 — External attested/runtime-bound evidence (strong independent)
- Examples: signed relay/runtime receipts, verifier-side signatures, cross-process attestations.
- Rule: always counts as independent when signature/identity check passes.

## Minimal deterministic policy upgrade
1. Replace boolean `independentSourcePresent` with:
   - `independenceScore` (0..N),
   - `maxEvidenceTier` (0|1|2),
   - `independentEvidenceQualified` (bool).
2. Completeness gate becomes:
   - required fields present,
   - `independenceScore >= 1`,
   - `maxEvidenceTier >= 1` (short term), upgradeable to `>=2` once strong attestations are available.
3. Emit `evidenceTierBreakdown` array for auditability.

## Suggested acceptance criteria (next implementation)
- Fixture A: Tier-0-only sources fail (`independentEvidenceQualified=false`, `failsClosed=true`).
- Fixture B: Tier-0 + valid Tier-1 source passes short-term gate.
- Fixture C: malformed Tier-2 signature does not count as independent.

## Why this improves honesty
- Prevents token “independent” labels from inflating confidence.
- Makes future migration to stronger attested evidence explicit and measurable.
- Preserves one-battle discipline while raising authenticity signal quality at artifact level.
